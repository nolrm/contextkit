# Context Budget

Use this command to decide which standards files to load and in what order for your current task. Context windows have finite attention — load what matters, skip what doesn't.

## Priority Order

Load in this sequence:

1. **Always load first:** `code-style.md`, `testing.md`
   - Highest signal. Relevant to almost every task.

2. **Load at session start:** `ai-guidelines.md`
   - Behaviour rules. Read once per session.

3. **Load on demand — architecture decisions only:** `architecture.md`
   - Only when making structural changes, adding new modules, or choosing patterns.

4. **Load on demand — process questions only:** `workflows.md`
   - Only when the task involves git flow, release process, or team conventions.

5. **Load selectively — match to file type being edited:**
   - `code-style/typescript-style.md` → TypeScript files only
   - `code-style/css-style.md` → CSS/SCSS files only
   - `code-style/javascript-style.md` → JS files only
   - Do not load all granular files at once.

6. **Skip unless task involves domain terms:** `glossary.md`

## Decision Tree

```
Are you writing code?
  → Yes: load code-style.md
  → Writing tests too? → load testing.md
  → Making architecture decisions? → load architecture.md

Are you unsure about a process step?
  → load workflows.md

Are you working on TypeScript files specifically?
  → load code-style/typescript-style.md (not the full code-style.md granular set)
```

## Length Warning

If a standards file exceeds 200 lines, read only the sections relevant to your task. Long files loaded in full compete for attention and reduce the effective signal of each section.

## Fresh Install Note

If standards files contain placeholder text (`Content will be generated by /analyze`), they have no useful content yet. Skip them and proceed with your best judgment until `/analyze` has been run.
