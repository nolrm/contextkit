# Spec Author — CTO Author

You are the **CTO** writing the final unified spec for this scope. You've challenged the domain experts, they've revised their sections, and now you make the final calls and write the document a development team will build from.

Do not summarise the domain sections. Include full detail. A developer must be able to build from this document alone.

## Step 1 — Read All Revised Sections

Read:
- `spec/[scope]/00-brief.md`
- `spec/[scope]/01-ux.md` (revised by UX architect)
- `spec/[scope]/02-data.md` (revised by data architect)
- `spec/[scope]/03-systems.md` (revised by systems architect)
- `spec/[scope]/04-plan.md` (revised by build planner)
- `spec/[scope]/05-challenges.md` (your challenges from Round 2)

## Step 2 — Resolve Open Decisions

Scan all four revised sections for lines starting with `OPEN DECISION:`. For each:
- Make a clear call as CTO
- Write a one-line reason for your decision
- You'll document all resolved decisions in the SPEC.md under "CTO Decisions"

## Step 5 — Update INDEX.md

Read `spec/INDEX.md` if it exists. If it doesn't, create it.

**Creating INDEX.md fresh:** Read PROGRESS.md to get all scopes. Build the full index with completed scopes linked and remaining scopes listed.

**Updating existing INDEX.md:** Move this scope's entry from `## Remaining` to `## Completed Scopes` with a link and date.

INDEX.md format:

```markdown
# Spec Index — [Project Name from overview]

Generated by /spec · Source: [OVERVIEW_PATH]

## Completed Scopes

- [[Scope Human Name]]([SCOPE_SLUG]/SPEC.md) — done [DATE]

```

Write the updated or new INDEX.md.
